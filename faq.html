❓ 常見問答

🧭 活動參加與流程相關

  • Q：什麼是 Hello, Godot! Game Jam？
    A：這是一場限定使用 Godot 引擎的遊戲開發活動，在指定期間內，依據主題設計並完成一款小遊戲。
  • Q：一定要填寫報名表才能參加嗎？
    A：並非必要，但為了確認參加者人數、協助分組與通知後續活動資訊，請務必填寫報名表。
  • Q：概念發表 是什麼？
    A：用你熟悉的工具（例如 PowerPoint、Word），分享組內的討論過程、發想方向等。即使還未決定明確方向也沒關係。
  • Q：DEMO發表 是什麼？
    A：DEMO發表 是活動中期的進度展示，你可以分享目前的遊戲開發狀況，例如初步完成的遊戲畫面、玩法系統或操作機制。
  • Q：中間發表後還能修改遊戲嗎？
    A：可以。活動中間的發表只是進度分享，之後仍可大幅調整內容與方向。
  • Q：沒有參加中間發表會有影響嗎？
    A：不會影響，主要目的是鼓勵大家參與、互動交流，同時也是提醒參加者此時應該有一定進度，避免中途放棄。

🛠️ 技術與工具相關

  • Q：Godot 是什麼？
    A：Godot 是一款開源、免費的遊戲引擎，適合製作 2D 和 3D 遊戲，支援多平台匯出。
  • Q：我可以使用哪個版本的 Godot？
    A：推薦下載使用最新的 4.4 穩定版。
  • Q：Godot 支援 C# 嗎？
    A：支援，但目前在 Web 匯出上有限制。如果要匯出 Web版本 則建議使用 GDScript，或考慮改用 Godot 3。

👥 組隊與社群參與

  • Q：可以單人參加或找認識的人組隊嗎？
    A：可以，也歡迎事先組隊或在 Discord 群組中尋找隊友。
  • Q：沒有認識的人可以怎麼組隊？
    A：填寫報名表單時，會依照需求協助安排，在活動前公告組別。
  • Q：報名表的「希望與同地區組隊」是什麼意思？
    A：便於安排可能想要面對面交流的參加者，盡量讓你與同地區的人同組。

🎮 成果與版權相關

  • Q：作品完成後如何提交？
    A：上傳至 itch.io 的 Game Jam 頁面。
  • Q：會有評分環節嗎？
    A：改為「投票制」，每個人可以在 Google 表單選擇自己喜歡的遊戲。
  • Q：作品的著作權屬於誰？
    A：著作權歸參加者所有，主辦僅以宣傳用途展示與分享，並不收費。
